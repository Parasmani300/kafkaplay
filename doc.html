Kafka Play Documentation

Getting Started

Welcome to Kafka Play! This documentation will guide you through the installation and usage of the app.

Installation

To install Kafka Play, follow these steps:

$ Download the zip file
$ Extract file to desired location
$ Double click on kafka-play-1.0.0 Setup.exe
$ Wait for installation to comlplete
$ Once installation is done. Navigate to C:\Users\${username}\AppData\Local\kafka_play
$ Send the .exe file to desktop

Usage

Once you have Kafka Play up and running, here are some common usage scenarios:

  • Producing Messages: Learn how to use Kafka Play to produce messages in Kafka.
  • Consuming Messages: Understand how to consume messages from Kafka topics using Kafka Play.
  • Kafka Administration: Perform administrative tasks like creating topics, resetting offsets, and more.

Usage

Produce Message

To produce a message using Kafka Play, use the following steps:

  1. Launch the app
  2. Select the produce option from the select menu above
  3. Give the broker url in the input box above e.g localhost:9092 or abc.com:9092
  4. Go to auth section if you are connecting to cloud cluster tick the cloud cluster option and specify the neccesary credentials, if connecting to localhost:9092, no need to tick or provide any auth credentials.
  5. Move to params section, Enter the topic and key (both mandatory) to send the message
  6. Lastly, Enter the message, if you want to send json check the JSON radio button if you want to send string check the string radio button, and at the end press Send

If the provided credentials are valid, then the message will be produced. The response can be seen in the response body section

Consume Message

To consume a message using Kafka Play, use the following steps:

  1. Launch the app
  2. Select the consume option (Selected by default)
  3. Enter the auth and broker url same as producer. follow step 3 and 4 of producer doc
  4. Enter the topic,key,groupId,auto offset reset ploicy
  5. Press send button. If the message exists it will be returned other wise empty array will print

*** If you do not have any consumer group create a new by entering some new name
